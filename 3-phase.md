# Phase 3: Data Sources - Streamlined Data Connection & Management

## Objective

Transform the Data Sources page (Integrations) into a user-friendly data connection hub that:
1. Makes it crystal clear how to get data into the system
2. Provides step-by-step guidance for each data source type
3. Shows real-time sync status and health
4. Handles errors gracefully with actionable solutions
5. Supports both one-time uploads and continuous syncing

---

## Current State Analysis

### What Exists
- Integration catalog (10 types: Google Sheets, Firebase, Supabase, etc.)
- BaseAdapter interface with standard methods
- Add Data Source modal with type-specific forms
- Status tracking (connected, error, syncing, paused)
- Sync strategies (manual, scheduled, realtime, webhook)
- Integration context for state management

### Critical Problems

| Problem | User Impact | Priority |
|---------|-------------|----------|
| No clear "Start Here" guidance | Users don't know which source to pick | Critical |
| Upload buried in separate page | Disconnect between upload and integrations | Critical |
| No game association | Data uploaded but not linked to any game | Critical |
| Complex configuration forms | Users abandon setup mid-way | High |
| No connection testing | Users don't know if config is correct | High |
| Error messages are technical | Users can't self-resolve issues | High |
| No data preview before import | Users import wrong data | Medium |
